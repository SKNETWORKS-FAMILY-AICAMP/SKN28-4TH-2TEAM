## 1. 인수인계 5개 의심 버그 — 검증 결과
| 주장 | 판정 | 근거 |
|---|---|---|
| 질문 2회 분석(P1-a) | 무해 | analyze는 순수함수, 결과 동일. 격하 |
| 근거검사 우회(P1-b) | **무해→drop** | 제거해도 no-op(검색기가 이미 dept 필터), 실제 답변 양호 |
| 분류기 비대(P2) | 사실 | 리팩터링 가치 O (예정: S4) |
| 선착순 분류(P3) | **사실** | 룰 *순서* 의존(위치 아님). 답변레벨 버그 A/B로 확인 |

## 6. 작업 방식 교훈
1. 인수인계 진단은 **증상과 원인을 분리 검증** (P3는 위치설→룰순서로 정정).
2. **자명한 한 줄 수정이 no-op일 수 있다** (P1-b: 빈 매핑 fallback).
3. **버그 판정은 중간 신호 말고 최종 답변으로** (추론→no-op 결론이 3회 반복됨).
