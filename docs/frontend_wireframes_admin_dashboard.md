# 화면설계서 — 관리자 (수집 데이터 현황)

> RAG 챗봇이 검색하는 수집 데이터셋의 규모·구성을 한눈에 보는 관리자 대시보드. 모든 수치는 실제 적재 레코드에서 집계.

| 라우트 | 접근 | 진입 | 데이터 출처 |
|---|---|---|---|
| `/admin-stats/` | 관리자 전용 | 채팅 헤더 `관리자` 버튼(`role==='admin'`) | `window.KB_DATA` 집계 |

---

## 1. 관리자 대시보드

![관리자 대시보드 번호 표기](images/screens/annotated/admin.png)

| 번호 | 화면 요소 | 설명 / 동작 |
|:--:|---|---|
| ① | 크롤링 기준 뱃지 | 데이터 수집 기준일("2026-06-17 크롤링 기준") |
| ② | 채팅으로 버튼 | 채팅 화면 복귀 |
| ③ | KPI 카드 4종 | 단과대학 · 학과·대학원 · 수집 교과목 · 교수·연구진 (천 단위 콤마) |
| ④ | 데이터 구성(가로 막대) | 교과목·교수·세미나·입학 유형별 건수, 진입 시 막대 애니메이션 |
| ⑤ | 단과대학별 비중(도넛) | SVG 도넛 + 중앙 총 레코드 + 범례(학과명·건수·%) |
| ⑥ | 학과별 수집 현황(표) | 학과·대학원 / 단과대학 / 교과목 / 교수 / 세미나 / 입학·장학 |

---

## 2. 집계 지표 정의

| 지표 | 계산 |
|---|---|
| 단과대학 / 학과·대학원 | 수집 단과대학 수 / 학과 마스터 수 |
| 수집 교과목 / 교수·연구진 | `KB_DATA.courses` / `.people` 레코드 수 |
| 데이터 구성(막대) | 레코드 유형별 건수, 최댓값 기준 비율로 막대 길이 |
| 단과대학 비중(도넛) | 단과대학별 총 레코드 비중(%) |
| 학과별 표 | 단과대학·교과목·교수·세미나·입학을 학과 단위 합산·정렬 |

> 현재 수치는 정적 데이터셋(`kb_data.js`) 기준입니다. **운영 통계(질문 로그·미응답·답변 피드백)는 Django 백엔드 연동 시 확장**할 영역입니다.
